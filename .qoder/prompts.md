# Prompt Engineering Guide (Qoder Adaptation)
Библиотека шаблонов промптов. Используй эту структуру для точных запросов к агенту.

## Формула идеального промпта
[РОЛЬ] + [КОНТЕКСТ/ФАЙЛЫ] + [ЗАДАЧА] + [ОГРАНИЧЕНИЯ] + [ФОРМАТ ВЫВОДА]

## Шаблоны

### 1. Реализация новой фичи
Роль: Ты старший Flutter-разработчик.
Контекст: Работаю в файлах @{{FILES}}. Архитектура описана в context.md.
Задача: Реализуй {{TASK}}.
Ограничения:
- Сначала предложи план из 3-5 шагов.
- Не меняй существующий код без пояснения.
- Соблюдай DESIGN.md и правила безопасности.
  Вывод: План -> Код по файлам -> Команды для терминала.

### 2. Интеграция внешнего API
Роль: Ты эксперт по сетевым запросам и аутентификации.
Контекст: Подключаю API {{API_NAME}}. Документация: {{URL}}.
Задача: Напиши клиент с методами: {{METHODS}}.
Ограничения:
- Используй MCP fetch для проверки документации.
- Таймауты, retry при 429/5xx, кастомные исключения.
- Секреты только через dotenv.
  Вывод: Схема запроса/ответа -> Код клиента -> Пример использования.

### 3. Поиск и фикс бага
Роль: Ты отладчик Flutter/Dart.
Контекст: Ошибка: "{{ERROR_MESSAGE}}". Стек: {{STACK_TRACE}}.
Задача: Найди причину и предложи исправление.
Ограничения:
- Не угадывай. Сначала проанализируй лог и код.
- Предложи минимальную правку.
- Укажи, как воспроизвести и проверить фикс.
  Вывод: Причина -> Файл и строка -> Патч -> Проверка.

### 4. Ревью и рефакторинг
Роль: Ты код-ревьюер со строгими стандартами.
Контекст: Файл: @{{FILE}}.
Задача: Проведи ревью и предложи рефакторинг.
Ограничения:
- Проверь: типы, null-safety, дубли, хардкод, производительность.
- Оцени по шкале 1-10. Дай конкретные правки.
- Не меняй бизнес-логику без причины.
  Вывод: Оценка -> Список проблем -> Готовый исправленный код.

### 5. Генерация UI-компонента
Роль: Ты UI-инженер Flutter.
Контекст: Стиль: DESIGN.md. Компонент: {{COMPONENT_NAME}}.
Задача: Сгенерируй виджет со всеми состояниями.
Ограничения:
- Используй константы из theme/.
- Покрой: idle, loading, success, error.
- Анимации только плавные (200ms).
  Вывод: Структура виджета -> Код -> Инструкция по интеграции.

## Правила использования
1. Всегда начинай запрос с выбора шаблона.
2. Заменяй {{...}} на конкретные данные.
3. Прикрепляй файлы через @ или указывай путь.
4. Требуй план перед кодом.
5. Запрети "молчаливые" правки. Каждое изменение должно быть объяснено.
