# 📱 Проект: Kling Image Generator (Flutter)

## 🎯 Цель
Разработать Flutter-приложение для генерации изображений через Kling API по текстовому промпту (с опциональной загрузкой исходного изображения).

## 🛠 Стек
- Язык: Dart (Flutter)
- Архитектура: Простая слоистая, готовность к MVVM/Clean
- State Management: Пока без тяжёлых фреймворков, базовые `ChangeNotifier` или `ValueNotifier`
- HTTP: `http` или `dio`
- Минимальный SDK: Android 21+, iOS 13+

## 🔑 API: Kling AI
- Документация: https://kling.ai/document-api/quickStart/productIntroduction/overview
- Базовый URL: `https://api.klingai.com`
- Аутентификация: JWT через Access Key + Secret Key (подпись запросов по спецификации)
- Ключи (⚠️ НЕ КОММИТИТЬ В GIT!):
  - ACCESS_KEY: `ADBBETNd4mYm9ktTbkJamYQJ8EhbPPCLJ`
  - SECRET_KEY: `mnTYffRb3CbmdMQadHFF3dgrbQrDrLrP`
- Хранение ключей: Только локально в `.env` или через `flutter_dotenv`, в коде — через `String.fromEnvironment`.

## ⚠️ Критические ограничения
1. Никогда не хардкодить ключи в исходный код.
2. Генерация асинхронная (30–120 сек) → требуется механизм поллинга статуса задачи.
3. Обязательная обработка ошибок: таймауты, лимиты, невалидные промпты, сетевые сбои.
4. Все сетевые запросы должны иметь явные таймауты.

## 🤖 Правила для AI-агента
- Перед написанием кода всегда согласовывай структуру файлов.
- Предлагай код небольшими, проверяемыми блоками.
- Не меняй существующий код без пояснения причин.
- При работе с API сначала показывай схему запроса/ответа, затем Dart-реализацию.
- Все секреты выноси в `.env`/`flutter_dotenv`.
